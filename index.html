Bottleneck Workarounds
4 PATCHES APPLIED
BOTTLENECK 500MB DB still a ceiling
Partition + Archive Cold Data
Use pg_partitioning on high-volume tables by date. Offload rows older than 90 days to Cloudflare R2 as compressed JSON via a nightly n8n export. DB stays lean, history stays accessible.
pg_partitioning FREE Infinite archive
BOTTLENECK 512MB split across services
Service Isolation on Railway
Pin n8n to its own Railway service with a hard memory limit. App server claims full 512MB uncontested. n8n only wakes for automation runs idle RAM usage drops to ~80MB.
Railway Services NO EXTRA COST Full app RAM
BOTTLENECK 10k Redis commands/day
Batch Writes via n8n Queue
Instead of per-request Redis writes, buffer state in-memory and flush in batches every 30s via an n8n interval trigger. Cuts Redis command volume by 80–90% with zero loss of data integrity.
n8n Batching FREE 10× efficiency
BOTTLENECK No native observability
Self-Rolled Health Monitor
Combine Supabase pg_cron + Umami event tracking + n8n alerting. Ping endpoints every 5 min, log response times to Supabase, alert via email on anomaly. Full ops visibility at zero cost.
pg_cron Umami FREE
